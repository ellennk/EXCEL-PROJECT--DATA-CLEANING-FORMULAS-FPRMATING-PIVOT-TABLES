# EXCEL-PROJECT--DATA-CLEANING/PRE-PROCESSING

### PROJECT OVERVIEW
This project covers the line between a workable data and a raw one.It spells out how a raw data can be cleaned well enough to be used for further analysis. The focus is on systematically identifying and correcting errors- such as duplicates,missing values and inconsistent formats.

   The aim of cleaning data is to transform raw data into a reliable, usable format.This process inreases accuracy, efficiency and relaibility,preventing costly business errors and enabling a more precised data.


### KEY OJECTIVES
1.Ensuring Accuracy and Consistency: This removes "noise" like typos, duplicate entries, and conflicting formats (e.g., mixing MM/DD and DD/MM dates) so the information reflects reality.

2.Facilitating Model Readiness: Most analytical tools and machine learning algorithms cannot process "dirty" data; cleaning structures the dataset into a standardized format that systems can actually read and compute.

3.Minimizing "Garbage In, Garbage Out": By stripping away irrelevant or incorrect values, you ensure that the final analysis leads to trustworthy business decisions rather than misleading or costly errors.

   ### KEY ACTIONS TAKEN
   -- Data Cleaning/ Pre-processing

1. Remove Duplicates & Irrelevant Data :All duplicate observations-Rows and Columns were successfully deleted.
  
2.Fix Structural Errors: Establish a Standardized naming conventions for each column.For instance,with this project the standard data format implemented is DD-MMM-YYYY and the  standard cases used is the "Proper Approach". Additionally,typos were fixed and align capitalization to resolve inconsistencies.

3.Address Missing:Options on missing items were to either to permanently deleted, replace or ignored. For the purposes of this project all blank rows were filled. Say blank doctors were replaced with unasssigned whereas missing  diagnosis were replaced with unknown.

4. Data Validation: An Intentional systematic checks used to ensure data that only certain data were allowed for specific columns. In this project's case, the gender colums was only made to contain 3 genders-Male,Female and others.Additionally, only a specifit spacing was allowed using the "Trim" function

### CONCLUSION
Data cleaning is the essential bridge between raw, chaotic information and actionable insights. Its purpose is to transform "noisy" data—riddled with gaps, duplicates, and errors—into a high-quality asset that ensures your conclusions are accurate and your algorithms are reliable.

Ultimately, data cleaning is an inevitable reality of the analytical process; because real-world data is inherently messy, you cannot bypass this stage without risking "garbage in, garbage out" results. It is not just a preliminary chore, but the foundational step that determines the integrity of every decision made from your data.
